## Coverage class definitions (present-state only)

| Class | Meaning |
|--------|---------|
| **Covered** | Existing Vitest and/or Playwright tests clearly exercise the surface or module (behavioral and/or copy-contract tests tied to that surface). |
| **Partially covered** | Automated linkage exists but is indirect, narrow, or split across files such that the full surface is not uniformly exercised in one place. |
| **Uncovered** | No dedicated automated test artifact reasonably tied to this surface; any linkage is incidental or type-level only. |
