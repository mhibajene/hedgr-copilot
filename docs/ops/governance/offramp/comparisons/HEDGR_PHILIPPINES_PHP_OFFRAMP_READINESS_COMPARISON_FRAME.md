## 2. Strategic context

The Kenya / KES / M-Pesa thesis stress-test path has produced valuable friction intelligence.

That friction is not treated as failure. It is treated as product and governance intelligence.

Because Kenya is friction-dense, Hedgr needs a comparator before overfitting the Class B readiness model to one difficult African mobile-money path.

The Philippines is proposed as a readiness-optimized benchmark because public source classes appear to offer a potentially more observable digital asset, regulated-provider, e-wallet, bank, and retail-payment-system frame than Kenya currently offers.

This artifact does not conclude that the Philippines is objectively superior, easier, selected, viable, legally available, operationally available, provider-ready, or Class B-ready.

## 10. Philippines friction hypotheses

### 10.1 Lower-friction hypothesis

The lower-friction hypothesis is that a Philippines / PHP frame may expose potentially more observable public regulated-provider, e-wallet, bank-account, and retail-payment-system source classes than the Kenya / KES / M-Pesa path, potentially reducing some endpoint-discovery and MNO-prefunding uncertainty.

This is not assumed.
It must be tested only if separately authorized.

If tested later, this hypothesis must still preserve that lower apparent friction does not mean selected, better, easier, viable, ready, legally available, operationally available, provider-ready, endpoint-approved, or Class B-ready.

### 10.2 Similar-friction hypothesis

The similar-friction hypothesis is that Philippines / PHP may still expose the same core Class B questions as Kenya: provider-of-record responsibility, stablecoin conversion owner, legal entity and licensing perimeter, same-name validation, KYC / KYB / AML / sanctions allocation, local endpoint limits, settlement finality, fees / FX / spread, failed payout handling, refunds, complaint ownership, support handoff, reconciliation truth, audit evidence, and incident stop conditions.

This is the likely default until evidence says otherwise.

This is not assumed.
It must be tested only if separately authorized.

### 10.3 Different-friction hypothesis

The different-friction hypothesis is that the Philippines / PHP path may not be lower-friction or similar-friction; it may shift friction away from MNO / mobile-money prefunding and into regulated-provider perimeter, payment-system applicability, e-wallet / bank endpoint support, business-account eligibility, same-name validation, provider-of-record reliance, or retail-payment finality.

This is not assumed.
It must be tested only if separately authorized.
