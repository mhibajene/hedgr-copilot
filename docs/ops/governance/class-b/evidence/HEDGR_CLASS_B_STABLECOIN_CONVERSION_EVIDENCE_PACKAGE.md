# HEDGR - Class B Stablecoin / Conversion Evidence Package

## 1. Status / Authority / Scope

**Status:** Evidence package skeleton; documentation-only.

**Authority:** Non-authoritative evidence-planning artifact subordinate to `docs/ops/HEDGR_STATUS.md`, `AGENTS.md`, accepted ADRs, doctrine, `EVID-B-001` through `EVID-B-009`, future qualified legal / compliance review, and future vendor / provider evidence where applicable.

**Scope:** Defines required stablecoin / conversion evidence items, source expectations, review questions, blockers, freshness expectations, and downstream dependencies for future Class B consideration.

**Mode:** Evidence package scaffolding only; not evidence gathering, evidence acceptance, stablecoin approval, conversion approval, redemption approval, FX approval, fee / spread approval, custody approval, rail approval, treasury approval, legal advice, regulatory approval, or Class B readiness.

This package does not approve stablecoin acquisition, stablecoin holding, stablecoin transfer, conversion, redemption, FX execution, fee / spread model, quote model, slippage model, custody model, rail pathway, treasury operation, liquidity operation, settlement finality, ledger truth, accounting truth, external engagement, Class B execution, implementation, staging, live operations, deposits, withdrawals, custody, rails, treasury operations, Copilot execution, or customer fund movement.

This package does not provide legal advice or claim regulatory approval.

This package exists to instantiate the stablecoin / conversion evidence package from the evidence gathering plan and registry.

## 13. Downstream dependency effect

- **Fee / FX / spread evidence package:** must reflect conversion quote behavior, expiry, final value evidence, fees, spreads, slippage, disclosure constraints, and dispute handling.
- **Liquidity / withdrawal control evidence package:** cannot mature without redemption assumptions, withdrawal readiness, liquidity dependency, custody transition evidence, and rail settlement evidence.
- **Trust UX / disclosure evidence package:** must not claim converted, redeemed, stable, safe, guaranteed, best rate, no fees, no slippage, withdrawable, or final without accepted evidence.
- **Pilot Ops evidence package:** must reflect conversion owner, provider owner, manual review, quote expiry, failed conversion, reversal, dispute, pause, and escalation responsibilities.
- **Reconciliation evidence package:** must map stablecoin / conversion evidence to custody state, rail state, quote state, settlement state, audit receipts, and source-of-truth boundaries.
- **Support / dispute evidence package:** must define support constraints around failed, delayed, reversed, disputed, unsupported, stale-quote, and unresolved conversion / redemption states.
- **Audit / retention evidence package:** must capture quote evidence, conversion evidence, redemption evidence, custody evidence, rail evidence, support evidence, reconciliation evidence, and retention requirements.
- **ADR drafting readiness:** cannot proceed for stablecoin / conversion-sensitive ADRs until required evidence exists or blockers are explicitly recorded.
- **Implementation-proposal preflight:** cannot treat stablecoin / conversion questions as closed without accepted evidence.
