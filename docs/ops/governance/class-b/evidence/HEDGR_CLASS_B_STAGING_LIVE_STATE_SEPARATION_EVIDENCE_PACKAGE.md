# HEDGR - Class B Staging / Live-State Separation Evidence Package

## 1. Status / Authority / Scope

**Status:** Evidence package skeleton; documentation-only.

**Authority:** Non-authoritative evidence-planning artifact subordinate to `docs/ops/HEDGR_STATUS.md`, `AGENTS.md`, accepted ADRs, doctrine, `EVID-B-001` through `EVID-B-018`, `GOV-B-015`, future qualified legal / compliance review, and future operational / vendor evidence where applicable.

**Scope:** Defines required environment-state separation, staging / live-state boundary, data separation, customer-money separation, rail / custody environment, support / reconciliation / audit environment, and Copilot environment-boundary evidence items, source expectations, review questions, blockers, freshness expectations, and downstream dependencies for future Class B consideration.

**Mode:** Evidence package scaffolding only; not evidence gathering, evidence acceptance, environment approval, environment transition approval, sandbox approval, internal-test approval, live approval, legal advice, regulatory approval, ADR drafting readiness, implementation readiness, or Class B readiness.

This package exists solely to define evidence requirements. It does not approve environments, environment transitions, testing, readiness, implementation, sandbox use, internal testing, manual pilots, limited live pilots, production operations, Class B execution, deposits, withdrawals, custody, rails, stablecoin conversion, treasury operations, reconciliation operations, support operations, audit operations, Copilot execution, or customer fund movement.

This package does not provide legal advice or claim regulatory approval.

## 2. Purpose

This package exists to instantiate the Staging / Live-State Separation evidence package from the evidence gathering plan and registry.

It answers:

**What environment-state separation and state-boundary evidence must be gathered and reviewed before future Class B ADR drafting readiness or implementation-proposal preflight can be considered?**

**Are staging, sandbox, internal testing, manual pilot, limited live pilot, production, support, reconciliation, audit, Copilot, custody, rail, or customer-money environments approved for Hedgr?**

## 11. Downstream dependency effects

- **`GOV-B-015`:** Provides the upstream staging / live-state separation memo dependency, but this package does not accept evidence or approve any environment state.

- **Pilot Ops evidence package (`EVID-B-014`):** Requires pilot-candidate, internal-test, sandbox, customer-money exclusion, and state-boundary evidence before pilot-adjacent review can assess environment scope.
- **Reconciliation evidence package (`EVID-B-015`):** Requires reconciliation environment labels and state-boundary evidence before source-of-truth, exception, unresolved-state, ledger, or accounting questions can be evaluated.
- **Support / Dispute evidence package (`EVID-B-016`):** Requires support environment labels and customer-money boundary evidence before support-state or closure claims can be evaluated.
- **Incident Response / Operational Failure evidence package (`EVID-B-017`):** Requires environment labels, incident-state tags, and customer-money exclusion evidence before incident or operational-failure review can rely on environment state.
- **Audit / Retention evidence package (`EVID-B-018`):** Requires audit tags, data classification, access-boundary evidence, and evidence source pointers before record-management or preservation claims can be reviewed.
- **ADR drafting readiness:** Cannot proceed for staging / live-state separation, environment taxonomy, data separation, customer-money separation, rail / custody environment, support, reconciliation, audit, or Copilot environment boundaries until required evidence exists or blockers are explicitly recorded.
- **Implementation-proposal preflight:** Cannot treat staging, sandbox, internal-test, manual pilot, limited-live, production exclusion, data separation, customer-money separation, rail / custody environment, support, reconciliation, audit, or Copilot environment readiness as closed without accepted evidence.

These dependency effects identify potential relationships only. They do not create sequencing authority, readiness, or approval.
