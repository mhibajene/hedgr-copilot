## 4. What this artifact is pressure-testing

This micro-test evaluates how the preferred copy directions behave when placed into a small number of upper-shell arrangements.

In scope:

- balance-label treatment within the header
- trust-line placement and emphasis
- action-cluster proximity to the balance anchor
- activity section naming in flow with the upper shell

This is a layout micro-test only.

It is intended to answer:

- does the balance still anchor the surface first?
- does the trust line read as calm reassurance rather than system narration?
- do actions feel connected to the money anchor without becoming primary?
- does `Recent activity` read naturally in the current shell?
- does the upper shell still feel like trusted everyday money rather than portfolio management?

---

## 5. Evaluation criteria

### 5.1 Money-first hierarchy

The balance must remain the primary reading anchor.

### 5.2 Calm reassurance

Trust language must remain visible, but quieter than the balance.

### 5.3 Action subordination

Top-up / send actions should read as available extensions of the money anchor, not as competing focal points.

### 5.4 Everyday-money familiarity

The overall upper shell should feel closer to a trusted money account than to a portfolio tracker or system console.

### 5.5 Subtraction over invention

The test should improve clarity through tighter placement and restraint, not by adding new elements or conceptual layers.

---

## 6. Micro-test variants

### 6.1 Variant A — Tight money-anchor cluster

**Structure**

- balance card remains first
- trust line remains inside the balance card
- action cluster moves visually closer to the balance card
- `Recent activity` remains the first block below actions

**Read**

This keeps the strongest current shell logic intact while reducing the feeling that actions are floating as a separate layer.

**What it gets right**

- preserves money-first hierarchy
- improves money-to-action relationship
- keeps trust tied to the balance anchor
- requires minimal structural movement

**Primary risk**

- if spacing tightens too far, actions may begin to compete with the balance rather than extend from it

**Working assessment**

This is the strongest current centerline for layout testing because it improves continuity without reopening the shell.

---

### 6.2 Variant B — Embedded action adjacency

**Structure**

- balance and trust remain primary
- actions sit in a more visibly attached zone directly beneath the balance card
- `Recent activity` begins slightly lower than in Variant A to preserve hierarchy separation

**Read**

This creates a stronger link between money and action, but increases the chance that the header becomes more product-utility-forward than intended.

**What it gets right**

- clearer extension from balance to action
- stronger everyday transaction logic
- improved immediacy for common wallet actions

**Primary risk**

- may over-promote actions
- may reduce the calmness currently achieved by the shell
- can drift toward utility-wallet posture if not carefully restrained

**Working assessment**

Admissible as an upper-bound test, but should be treated as a tension detector rather than the default centerline.

---

### 6.3 Variant C — Looser trust separation

**Structure**

- balance remains first
- trust line becomes slightly more visually separated from the balance amount
- actions remain close to the card, but trust is given more breathing room as a secondary explanatory line
- `Recent activity` remains unchanged below

**Read**

This tests whether the shell becomes calmer and more premium when reassurance is allowed to breathe slightly more, rather than feeling embedded as a dense system line.

**What it gets right**

- can improve calmness
- can reduce system-authored density
- may make the header feel more composed

**Primary risk**

- trust may become too detached from the money anchor
- upper shell may lose some immediate coherence
- can make reassurance feel decorative rather than integral

**Working assessment**

Useful as a calibration variant, but weaker than Variant A unless it clearly improves readability without diluting the trust anchor.

---

## 7. Variant comparison readout

### 7.1 Strongest current centerline

**Variant A — Tight money-anchor cluster** is the strongest current layout direction.

Reason:

- it best preserves the governed shell
- it improves money-to-action continuity
- it supports the copy-refinement outcomes
- it creates gains through tightening rather than structural reinvention

### 7.2 Most useful upper-bound detector

**Variant B — Embedded action adjacency** is the best detector for action-layer overreach.

Reason:

- it reveals how easily the shell can drift from calm money account toward utility wallet
- it helps test the exact boundary where actions become too loud

### 7.3 Calibration variant

**Variant C — Looser trust separation** is useful only as a calmness calibration check.

Reason:

- it can improve breathing room
- but it risks weakening the trust-to-money relationship that currently gives the shell coherence

---

## 8. Key findings

- The header performs best when the balance remains unmistakably first and everything else reads as support.
- Trust phrasing should remain attached closely enough to the balance anchor to feel integral, not ornamental.
- The action cluster benefits from moving closer to the money anchor, but only within a clearly subordinate posture.
- `Recent activity` remains the most natural transition from upper-shell money state into the body content.
- The shell weakens when actions begin to read as the primary behavioral invitation.
- The shell also weakens when trust becomes too detached, technical, or system-authored.

---

## 9. Working recommendations

### 9.1 Layout centerline

Use **Variant A — Tight money-anchor cluster** as the current layout centerline for the next refinement step.

### 9.2 Action-layer rule

Allow actions to move closer to the balance anchor, but do not permit them to rival the balance in visual or conceptual priority.

### 9.3 Trust-line rule

Keep the trust line close enough to the balance to preserve a single money-first read, while maintaining calm phrasing and low visual pressure.

### 9.4 Section naming rule

Retain **`Recent activity`** as the current activity naming centerline unless a later in-context test produces a materially stronger governed read.
