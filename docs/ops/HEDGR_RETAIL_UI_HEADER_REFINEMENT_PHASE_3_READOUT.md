## 5. What Phase 3 was trying to resolve

The question in Phase 3 was no longer which shell to choose. The shell direction was already narrowed to money-first.

The working question was how to make that chosen shell read more naturally and trustworthily at the top of the screen through bounded refinement.

Core tensions under pressure:

- whether balance labeling should be reduced or retained for clarity
- how to preserve Hedgr trust signature without system-heavy narration
- how to replace less familiar naming with everyday-money language
- how to move actions closer to the money anchor without making them dominant
- how to preserve Family C visibility while keeping it subordinate

---

## 6. What Phase 3 clarified

- the money-first shell remains the correct governed base
- strongest gains now come from subtraction, not conceptual additions
- the balance must remain the unquestioned anchor of the upper shell
- trust performs best as a calm attached layer, not a monitoring narrative
- familiar everyday-money naming materially improves readability
- action proximity helps only when held within clear hierarchy restraint
