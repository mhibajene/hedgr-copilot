# Hedgr UX Constitution Addendum

## Treasury Allocation Model Integration

Version: 1.0

Status: Binding Extension to UX Constitution

---

## Context

The Hedgr UX Constitution establishes:

- Trust as primary interface
- Safety over speed
- Advisory, never directive
- Calm as a feature
- Growth must never compete with protection oai_citation:2‡Hedgr UX Constitution.md

This addendum encodes the Default Allocation Policy into the UX layer.

---

## 1. Allocation Must Feel Deliberate, Not Speculative

The product shall not present deposits as “earning.”

The interface must present deposits as:

“Allocated between Stability and Conservative Yield.”

Language guidelines:

- Use “Allocate” instead of “Earn.”
- Use “Stability Buffer” instead of “Idle.”
- Use “Conservative Yield” instead of “Max Yield.”

Yield is framed as enhancement within risk boundaries.

---

## 2. Stability Buffer Must Be Visible

The buffer percentage must be visible and understandable.

The user must always know:

- What portion is buffered
- What portion is deployed
- Why that split exists

Hiding the buffer violates “Risk Must Be Visible.”

---

## 3. Risk Shown Before Return

When displaying yield:

- Risk tier must be shown alongside it
- Exposure distribution must be accessible
- Liquidity buffer must be visible

APY must never appear in isolation.

---

## 4. Calm Response During Reallocation

When the system increases buffer or reduces yield exposure:

- The UI must frame it as protection
- Tone must be calm and transparent
- No alarmist language
- No urgency framing

Example tone:

“We’ve increased your Stability Buffer due to elevated volatility. Your principal remains protected.”

---

## 5. No Gamification of Yield

Explicitly banned:

- Celebratory animations for yield
- Leaderboards
- Daily streaks
- Aggressive reinvest prompts

Yield accumulation must feel steady, not exciting.

This aligns with:

> “Calm is a feature.”  oai_citation:3‡Hedgr UX Constitution.md
> 

---

## 6. Behavioral Outcome Target

After 12 months, a Hedgr user should unconsciously believe:

- Some capital is always buffered.
- Yield fluctuates, principal should not.
- Allocation is intentional.
- Risk management is normal.

UX must reinforce this consistently.

---

## 7. Design Review Rule

Any design that:

- Encourages full deployment
- Minimizes buffer visibility
- Prioritizes APY over allocation
- Increases urgency

Is a constitutional violation.

Such violations are treated as UX defects.

---

## Closing Principle

Hedgr’s allocation model is not a feature.

It is behavioral architecture.

Treasury discipline must be felt in the interface without being preached.
