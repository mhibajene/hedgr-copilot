## 1. Trust Is the Interface  
**Pillars: Security & Trust**

Trust is not a feature. It is the default state of the product.

We design so that users always know:
- Where their money is
- What their money is doing
- How they can exit

**Rules**
- Status is shown before action
- Risk is explicit, never implied

## 2. Safety Over Speed  
**Pillars: Security & Trust, Regulatory Awareness**

In volatile economies, speed without clarity is dangerous.

We design so that:
- Waiting is a valid outcome
- Slower flows are acceptable if they reduce mistakes
- No irreversible actions happen without clear confirmation

**Rule**
> If speed increases user risk, we slow down.

## 3. Advisory, Never Directive  
**Pillars: Education & Empowerment, User-Centricity**

Hedgr does not command. Hedgr advises.

We design so that:
- The Copilot recommends, never instructs
- Every recommendation explains *why*
- Silence is acceptable when action is unsafe

**Hard rule**
> â€œDo nothingâ€ must be a first-class UX path.

## 4. Simplicity Is a Moral Obligation  
**Pillar: Simplicity & UX**

Complexity transfers risk to the user. This is unacceptable.

We design so that:
- A first-time user can explain Hedgr to a friend
- No screen requires prior crypto or DeFi knowledge
- Every term earns its place

**Rules**
- Plain language over jargon
- Explanation before interaction
- Fewer screens over clever abstractions

> If it needs a glossary, the UX is incomplete.

---

## 5. Calm Is a Feature  
**Pillars: Simplicity & UX, Security & Trust**

Financial products should reduce stress, not amplify it.

We design so that:
- The interface feels stable even when markets are volatile
- Visuals signal safety, not excitement
- Users never feel rushed

**Explicit bans**
- Flashing indicators
- Confetti or celebration for yield
- Red/green emotional manipulation

> Hedgr should feel closer to a vault than a trading floor.
